1. Соберите отряд
    Есть три класса юнитов: маги, стрелки и пехотинцы
  2. Напишите скрипт для своей армии
    Код – чистый JS, исполняется последовательно для каждого юнита
  3. Нажмите "Готово!"
    Сражение между игроками начнется только после того как оба будут готовы
  • Очередность хода зависит от скорости юнита, быстрые ходят первыми
  • Победитель определяется либо после полной победы одной из сторон, либо по очкам после окончания выполнения кода
goTo(x: number, y: number)
Переход на клетку x, y
relativeGoTo(x: number, y: number)
Переход на клетку x, y относительно позиции юнита
для правого игрока перемещения влево будут с отрицательным x
goToEnemyAndHit(id: string)
Переход в сторону противника по id и попытка атаки, если это возможно
пехотинец атакует только если может дотянуться до противника
shoot(id: string)
Стрелковая атака противника по id
применимо только к стрелкам
spell(id: string)
Магическая атака противника по id
применимо только к магам
say(text: string)
Сказать что-нибудь text
attackRandom()
Берсерк! Случайный выбор юнита на карте и атака!
атакует всех кроме юнитов с таким же id
isShooter(): boolean
Проверка, является ли юнит стрелком
isMagician(): boolean
Проверка, является ли юнит магом
isInfantry(): boolean
Проверка, является ли юнит пехотинцем
is(id: string): boolean
Возвращает true если ID юнита равен id

        stepsCount = 10

  2. Цикл

  3. Функция
        function attack(id) {
        if (isShooter()) {
            shoot(id)
        }
        if (isInfantry()) {
            goToEnemyAndHit(id)
        }
    }

    attack('ie')

Дано:

  • У нас сть 2 юнита: CSS и PWA
  • У противника: CSS и $

id юнита написан в фиолетовом прямоугольнике у его ног

Если заглянуть в характеристики персонажей:

  • Стрелковые атаки будуи менее эффективны против стрелков ($)
  • Чтобы нанести больше урона стрелкам надо подойти ближе к цели
  • У нас есть один пехотинец (CSS) с ближней атакой

Как написать более-менее эффективный скрипт:

  • Нажать "Сгенерировать пример кода", чтобы сразу получить стартовый код для нашей армии
  • Подвести PWA ближе к противнику, чтобы наносить больше урона
  • Ближнюю атаку направить на $ а стрелковую на — CSS
Код выполняется всего один раз, за один ход юниты не могут дойти до персоанажа или полностью убить его за один выстрел потому накидывайте по-больше действий на юнитов
  1. Каждому свое

    Разделяйте логику юнитов по классу персонажа 

        if (isShooter()) {
        shoot('ie')
    }
    if (isMagician()) {
        spell('ie')
    }
    if (isInfantry()) {
        goToEnemyAndHit('ie')
    }

    или по id

        if (is('eval')) {
        spell('ie')
    }
  2. Стрелки – ближе, маги – дальше

    Чтобы достичь максимальной атаки:

        // стрелки должны стоять максиум за 5 клеток от противника
    if (isShooter()) {
        relativeGoTo(5, 0)
    }
    // маги – минимум за 10
    if (isMagician()) {
        relativeGoTo(-2, -1)
    }
  3. Слишком много действий

    Если юнит противника погиб, а действия на его атаку еще остались...

        for (i = 0; i < 10; i++) {
        goToEnemyAndHit('dart')
    }
    for (i = 0; i < 10; i++) {
        goToEnemyAndHit('css')
    }

    ...юнит будет пропускать ходы пока не дойдет до действий атаки еще живого противника, если они еще есть

  4. Списки противников

    Бывает так, что противник все время меняет юнитов

        ids = ['ie', '$', 'dart']

    ids.forEach(id => {
        shoot(id)
        shoot(id)
    })

    потому их проще записать в массив и менять вместе с ним
